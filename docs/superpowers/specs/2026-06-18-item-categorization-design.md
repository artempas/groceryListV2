# Категоризация товаров через эмбеддинги (OpenRouter)

**Дата:** 2026-06-18
**Статус:** согласован, готов к планированию реализации

## Цель

Автоматически распределять товары списка покупок по фиксированному набору
категорий и группировать их по категориям на странице списка. Категория
определяется по имени товара через эмбеддинги OpenRouter в момент добавления
товара (на сервере).

## Решения (итог брейншторминга)

- **Категории:** фиксированный список, заданный в коде.
- **Момент категоризации:** при добавлении товара, синхронно на сервере
  (внутри `POST /api/lists/[id]/items`).
- **Отображение:** непокупленные товары группируются под заголовками категорий;
  купленные остаются единым блоком внизу, как сейчас.
- **Fallback:** при ошибке API, отсутствии ключа, таймауте или низкой близости
  товар создаётся с `category = null` и показывается в группе «Без категории».
  Категоризация никогда не ломает добавление товара.
- **Подход:** статические предвычисленные эмбеддинги категорий + косинусное
  сравнение (вариант A). Без pgvector, без хранения векторов товаров.

## Список категорий

Фиксированный набор. Каждая категория — имя + эталонная фраза с примерами
товаров. Эталонная фраза задаёт категорию «на языке товаров»: голое абстрактное
имя («Бакалея», «Молочное») плохо притягивает конкретные товары при косинусном
сравнении, а фраза с примерами помещает вектор категории в то же семантическое
соседство, где лежат реальные товары.

| Категория | Эталонная фраза (примеры товаров) |
|---|---|
| Овощи и фрукты | картофель, помидоры, огурцы, яблоки, бананы, зелень, лук, морковь |
| Молочное и яйца | молоко, сыр, творог, йогурт, кефир, сметана, масло сливочное, яйца |
| Мясо и рыба | курица, говядина, свинина, фарш, рыба, колбаса, сосиски |
| Бакалея | крупа, рис, гречка, макароны, мука, сахар, соль, масло подсолнечное, консервы |
| Хлеб и выпечка | хлеб, батон, булочки, лаваш, багет |
| Напитки | вода, сок, чай, кофе, газировка, лимонад |
| Сладости и снеки | шоколад, печенье, конфеты, чипсы, орехи, мармелад |
| Замороженное | пельмени, мороженое, замороженные овощи, замороженная пицца |
| Бытовая химия и хозтовары | моющее средство, стиральный порошок, губки, мусорные пакеты, фольга |
| Гигиена и уход | мыло, шампунь, зубная паста, туалетная бумага, гель для душа |

«Прочее / Без категории» отдельной записью не задаётся — это `category = null`.

> Точные имена и формулировки эталонных фраз — единственный источник правды в
> `lib/categories.ts`. Таблица выше носит ориентировочный характер.

## Архитектура и модули

### `lib/categories.ts`
Экспортирует упорядоченный массив определений категорий:

```ts
interface CategoryDef {
  name: string        // отображаемое имя, оно же значение ListItem.category
  reference: string   // эталонная фраза с примерами товаров
}
export const CATEGORIES: CategoryDef[]
```

Порядок массива задаёт порядок групп в UI.

### `lib/embeddings.ts`
Тонкий клиент OpenRouter:

```ts
export async function embed(text: string): Promise<number[]>
```

- Эндпоинт: `https://openrouter.ai/api/v1/embeddings` (OpenAI-совместимый).
- Конфиг из env:
  - `OPENROUTER_API_KEY` — обязателен.
  - `OPENROUTER_EMBEDDING_MODEL` — модель, дефолт `openai/text-embedding-3-small`.
- Бросает ошибку при отсутствии ключа или неуспешном ответе (обрабатывается выше).

### `lib/categorize.ts`
Главная функция категоризации:

```ts
export async function categorize(name: string): Promise<string | null>
```

- Эмбеддит `name` через `embed`.
- Считает косинусную близость с каждым предвычисленным вектором категории.
- Возвращает имя категории с максимальной близостью, если она ≥
  `CATEGORY_MATCH_THRESHOLD` (env, дефолт `0.30`); иначе `null`.
- Любая ошибка (нет ключа, сбой сети, таймаут) перехватывается и возвращает
  `null` — добавление товара не должно падать.

### `lib/category-vectors.json`
Предвычисленные эмбеддинги эталонных фраз: массив `{ name, vector }`.
Коммитится в репозиторий. Загружается `lib/categorize.ts`. При несоответствии
модели/размерности данных и текущего `OPENROUTER_EMBEDDING_MODEL` — это
ответственность разработчика (см. скрипт ниже); рантайм просто использует то,
что лежит в JSON.

### `scripts/build-category-embeddings.ts`
Одноразовый генератор `category-vectors.json`: проходит по `CATEGORIES`,
эмбеддит `reference` каждой через `embed`, пишет JSON. Запускается вручную при
изменении списка категорий или модели. Документируется в README.

### Схема Prisma
К модели `ListItem` добавляется поле:

```prisma
category String?
```

Миграция добавляет nullable-колонку; существующие строки получают `null`.

## Поток данных

1. `POST /api/lists/[id]/items`: после валидации `name` вызывает
   `categorize(name.trim())`.
2. Создаёт `ListItem` с `category` (строка или `null`) в одной транзакции с
   остальными полями. Ответ — тот же объект товара, что и сейчас, плюс `category`.
3. `GET /api/lists/[id]/items`: возвращает товары с полем `category` (никаких
   изменений в логике, поле просто появляется в выборке).
4. Клиент (`app/(app)/lists/[id]/page.tsx`):
   - Непокупленные товары группируются по `category`.
   - Группы рендерятся в порядке `CATEGORIES`; товары с `category = null` идут
     группой «Без категории» в конце непокупленных.
   - Внутри группы порядок — текущий (по `createdAt desc`).
   - Купленные товары остаются единым блоком «Куплено (N)» внизу, без группировки.
   - Заголовок группы использует тот же стиль, что текущий разделитель
     «Куплено (N)».

## Обработка ошибок и стоимость

- Категоризация изолирована: любой сбой → `category = null`, товар создаётся.
- Один embeddings-запрос на каждый добавленный товар.
- Векторы категорий считаются один раз скриптом — на стоимость рантайма не влияют.

## Тестирование

- **Юнит `lib/categorize.ts`:** косинус и выбор лучшей категории на замоканных
  векторах; срабатывание порога (низкая близость → `null`); `null` при ошибке
  `embed`/отсутствии ключа. `embed` мокается — реальные сетевые вызовы в тестах
  не выполняются.
- **Роут `POST /items`:** товар создаётся даже когда `categorize` бросает или
  возвращает `null`; поле `category` сохраняется и возвращается в ответе.

## Вне рамок v1

- **Бэкфилл существующих товаров.** Уже существующие в БД товары не
  перекатегоризируются — показываются в «Без категории», пока не пересозданы.
  Бэкфилл-скрипт может быть добавлен позже.
- Кэширование эмбеддингов по повторяющимся именам товаров.
- Пользовательские/динамические категории, фильтры и сворачивание групп.
- Хранение вектора товара в БД (pgvector).
