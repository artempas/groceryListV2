# Анимация «переезда» элемента в раздел «Куплено»

Дата: 2026-06-18

## Цель

При отметке элемента списка купленным он должен плавно «переезжать» вниз
в раздел «Куплено», а не мгновенно перепрыгивать. Эффект симметричен: снятие
отметки так же плавно возвращает элемент наверх в список покупок.

## Контекст

Страница списка — единый клиентский компонент
[`app/(app)/lists/[id]/page.tsx`](../../../app/(app)/lists/[id]/page.tsx).
Элементы хранятся одним массивом и разбиваются на два при рендере:

```ts
const unchecked = items?.filter((i) => i.checkedAt === null) ?? []
const checked = items?.filter((i) => i.checkedAt !== null) ?? []
```

Каждый массив рендерится отдельным `.map()` в `<ItemRow>`. Между секциями —
разделитель «Куплено (N)». Toggle реализован оптимистичной мутацией
(`toggleMutation`), которая мгновенно переписывает `checkedAt`, из-за чего
React перемещает строку без анимации.

## Подход

Используем библиотеку **framer-motion** (совместима с Next 14 / React 18;
компонент уже помечен `'use client'`).

### Изменения

1. **`ItemRow` → корневой `<motion.div layout>`.** Свойство `layout` включает
   FLIP-анимацию: при изменении позиции строки Framer Motion измеряет старое
   и новое положение и плавно прокатывает элемент на новое место. Стабильный
   `key={item.id}` (уже есть) гарантирует, что строка не размонтируется при
   переходе между секциями, а только меняет позицию.

2. **Обёртка списка в `<AnimatePresence>`** для корректной обработки
   появления/удаления элементов (добавление, swipe-to-delete).

3. **Разделитель «Куплено (N)»** делаем `layout`-элементом внутри
   `AnimatePresence`, чтобы его появление/исчезновение сдвигало раскладку
   плавно, а не скачком.

4. **Тайминги:** мягкий spring (`type: 'spring', stiffness: 500, damping: 40`)
   — быстрый отклик с лёгкой инерцией «доезда».

5. **Доступность:** уважаем `prefers-reduced-motion` — при включённой опции
   layout-анимации отключаются (через `MotionConfig reducedMotion="user"` или
   эквивалент).

### Что НЕ меняем

- Оптимистичные мутации (toggle / add / delete) и их логику.
- Swipe-to-delete внутри `ItemRow` (touch-обработчики и transform остаются;
  они применяются к внутреннему слою, layout-анимация — к внешнему
  `motion.div`, конфликта transform быть не должно — проверить при реализации).
- API и серверную логику.

## Риски / на что обратить внимание

- **Конфликт transform.** Swipe использует `transform: translateX(...)` на
  внутреннем `<div>`. Layout-анимация Framer Motion применяет transform к
  `motion.div`. Нужно убедиться, что свайп остаётся на внутреннем слое, а
  `layout` — на внешнем, чтобы анимации не перетирали друг друга.
- **Скачок при первом «Куплено».** Появление разделителя меняет высоту —
  layout-анимация должна сгладить это; проверить визуально.

## Тестирование

- Существующие тесты в `tests/` покрывают API-уровень; UI-теста для этой
  страницы нет.
- Проверить, что `npm test` остаётся зелёным (изменения чисто визуальные).
- Ручная проверка в браузере: отметка/снятие отметки, добавление, swipe-delete,
  поведение при `prefers-reduced-motion`.
